Integrating Firebase in Android: Authentication, Database, and Notifications

Published: August 10, 2025 | 12 min read

Firebase is Google’s powerful platform for building scalable Android apps with backend services like authentication, real-time database, and push notifications. In this beginner-friendly tutorial, we’ll guide you through integrating Firebase into your Android app using Kotlin. You’ll learn to set up Firebase, add email authentication, store data in Firestore, and send push notifications. Perfect for students and new developers!

Why Use Firebase?

Firebase simplifies backend development by offering ready-to-use services, saving you from building servers from scratch. Key features include:

  • Authentication: Support for email, Google, anonymous logins, and more.
  • Firestore: A scalable NoSQL database for real-time data syncing.
  • Cloud Messaging: Send push notifications to engage users.
  • Analytics: Track user behavior to improve your app.

This tutorial focuses on authentication and Firestore, with a brief look at push notifications. Let’s get started!

Step 1: Set Up Firebase in Your Project

Follow these steps to connect your Android app to Firebase:

  1. Create a Firebase Project: Go to Firebase Console and click "Add Project." Name it (e.g., "MidwayLearningApp") and enable Google Analytics (optional).
  2. Add Your Android App: In the Firebase Console, click "Add App" > Android. Enter your app’s package name (e.g., `com.example.midwaylearning`), app nickname, and SHA-1 key (optional, for authentication). Download the `google-services.json` file.
  3. Place Configuration File: Move `google-services.json` to your project’s `app/` directory in Android Studio.
  4. Add Firebase Dependencies: Update your Gradle files:
    • In `build.gradle` (project-level): 
      buildscript {
  dependencies {
    classpath 'com.google.gms:google-services:4.4.2' // Check for latest version
  }
}
    • In `build.gradle` (app-level): 
      plugins {
  id 'com.android.application'
  id 'kotlin-android'
  id 'com.google.gms.google-services'
}

dependencies {
  implementation 'com.google.firebase:firebase-auth:23.0.0'
  implementation 'com.google.firebase:firebase-firestore:25.0.0'
  implementation 'com.google.firebase:firebase-messaging:24.0.0'
}
    Sync the project with Gradle (click the elephant icon).
  5. Initialize Firebase: Ensure your `MainActivity` includes Firebase initialization (automatic with `google-services` plugin).

Tip: Check the Firebase Console for the latest dependency versions, as they update frequently.

Step 2: Add Email Authentication

Firebase Authentication supports email/password logins. First, enable it in the Firebase Console under Authentication > Sign-in method > Email/Password. Then, add a login UI (e.g., from our UI Design tutorial) and implement login logic.

package com.example.midwaylearning

import android.os.Bundle
import android.util.Log
import androidx.activity.ComponentActivity
import androidx.activity.compose.setContent
import androidx.compose.foundation.layout.*
import androidx.compose.material3.*
import androidx.compose.runtime.*
import androidx.compose.ui.Modifier
import androidx.compose.ui.unit.dp
import com.google.firebase.auth.FirebaseAuth
import com.example.midwaylearning.ui.theme.MidwayLearningTheme

class MainActivity : ComponentActivity() {
    private lateinit var auth: FirebaseAuth

    override fun onCreate(savedInstanceState: Bundle?) {
        super.onCreate(savedInstanceState)
        auth = FirebaseAuth.getInstance()
        setContent {
            MidwayLearningTheme {
                Surface(
                    modifier = Modifier.fillMaxSize(),
                    color = MaterialTheme.colorScheme.background
                ) {
                    LoginScreen(auth)
                }
            }
        }
    }
}

@Composable
fun LoginScreen(auth: FirebaseAuth) {
    var email by remember { mutableStateOf("") }
    var password by remember { mutableStateOf("") }
    var message by remember { mutableStateOf("") }

    Column(
        modifier = Modifier
            .fillMaxSize()
            .padding(16.dp),
        horizontalAlignment = Alignment.CenterHorizontally
    ) {
        Text(
            text = "Login to Midway Learning",
            style = MaterialTheme.typography.headlineMedium,
            modifier = Modifier.padding(bottom = 16.dp)
        )
        OutlinedTextField(
            value = email,
            onValueChange = { email = it },
            label = { Text("Email") },
            modifier = Modifier.fillMaxWidth()
        )
        OutlinedTextField(
            value = password,
            onValueChange = { password = it },
            label = { Text("Password") },
            modifier = Modifier.fillMaxWidth()
        )
        Button(
            onClick = {
                auth.signInWithEmailAndPassword(email, password)
                    .addOnCompleteListener { task ->
                        message = if (task.isSuccessful) {
                            "Sign-in successful!"
                        } else {
                            "Sign-in failed: ${task.exception?.message}"
                        }
                    }
            },
            modifier = Modifier.padding(top = 16.dp)
        ) {
            Text("Login")
        }
        Text(message, modifier = Modifier.padding(top = 8.dp))
    }
}

This code creates a login UI with Jetpack Compose and handles email authentication. Test with a valid email/password after registering a user in Firebase.

Step 3: Store Data with Firestore

Firestore is a NoSQL database for real-time data syncing. Enable it in the Firebase Console under Firestore Database > Create Database (use Test Mode for development). Below is an example of saving user data after login.

import com.google.firebase.firestore.FirebaseFirestore

val db = FirebaseFirestore.getInstance()
val user = hashMapOf(
    "username" to "JohnDoe",
    "email" to auth.currentUser?.email
)

db.collection("users")
    .document(auth.currentUser?.uid ?: "unknown")
    .set(user)
    .addOnSuccessListener {
        Log.d("Firestore", "User data saved")
    }
    .addOnFailureListener { e ->
        Log.e("Firestore", "Error saving data: ${e.message}")
    }

Add this code after a successful login to store user data. Retrieve it later using `db.collection("users").get()`.

Step 4: Send Push Notifications

Firebase Cloud Messaging (FCM) enables push notifications. Enable it in the Firebase Console and send a test notification via Cloud Messaging > Send Test Message. Add the following to handle notifications in your app:

import com.google.firebase.messaging.FirebaseMessagingService
import com.google.firebase.messaging.RemoteMessage

class MyFirebaseMessagingService : FirebaseMessagingService() {
    override fun onMessageReceived(message: RemoteMessage) {
        super.onMessageReceived(message)
        Log.d("FCM", "Notification: ${message.notification?.body}")
    }
}

Declare the service in `AndroidManifest.xml`:

<service
    android:name=".MyFirebaseMessagingService"
    android:exported="false">
    <intent-filter>
        <action android:name="com.google.firebase.MESSAGING_EVENT"/>
    </intent-filter>
</service>

Test by sending a notification from the Firebase Console.

Troubleshooting Common Issues

  • Authentication Errors: Ensure Email/Password is enabled in Firebase Console.
  • Firestore Fails: Check Firestore rules and network connectivity.
  • Gradle Sync Issues: Verify internet connection and update to latest dependency versions.
  • Notifications Not Received: Confirm FCM setup and device token registration.

Refer to Firebase’s official documentation or contact us for help.
