Kotlin Basics: Your First Step in Android Development

Published: August 25, 2025 | 10 min read

Kotlin is the preferred programming language for Android development, endorsed by Google for its modern, concise, and safe features. It’s interoperable with Java, beginner-friendly, and reduces common errors like null pointer exceptions. In this tutorial, we’ll cover Kotlin basics—variables, functions, control flow, and null safety—with practical examples to help you start coding Android apps.

Why Learn Kotlin?

Kotlin simplifies Android development with its modern syntax and safety features. Key benefits include:

  • Null Safety: Prevents null pointer crashes, a common issue in Java.
  • Concise Syntax: Reduces boilerplate code compared to Java.
  • Interoperability: Seamlessly works with existing Java libraries and Android code.
  • Community Support: Backed by Google and JetBrains, with a growing ecosystem.

This tutorial assumes you have Android Studio installed (see our Build Your First Android App tutorial). Let’s get started!

Step 1: Set Up Your Environment

To code in Kotlin, you need:

  • Android Studio: The official IDE for Android, with built-in Kotlin support (version Koala or later, 2025).
  • Kotlin Plugin: Included by default in Android Studio.
  • Optional: IntelliJ IDEA (by JetBrains) for non-Android Kotlin projects.

Create a new Kotlin project in Android Studio: File > New > New Project > Empty Activity, select Kotlin as the language. Alternatively, use the Kotlin Playground (play.kotlinlang.org) for quick experiments.

Step 2: Understand Basic Kotlin Syntax

Let’s start with a simple "Hello, World" program:

fun main() {
    val name = "Android"
    println("Hello, $name!")
}

Key Elements:

  • `fun`: Defines a function (e.g., `main` is the entry point).
  • `val`: Immutable variable (cannot be reassigned).
  • `println`: Outputs text to the console.
  • `$name`: String interpolation for embedding variables.

Run this in Android Studio’s Kotlin REPL (Tools > Kotlin > Kotlin REPL) or a new Kotlin file (`main.kt`). Output: `Hello, Android!`

Step 3: Work with Variables and Data Types

Kotlin has two variable types: `val` (immutable) and `var` (mutable). Common data types include `String`, `Int`, `Double`, and `Boolean`.

fun main() {
    val name = "Midway Learning" // Immutable
    var age = 25 // Mutable
    age = 26 // OK
    // name = "New Name" // Error: val cannot be reassigned

    val score: Double = 95.5
    val isActive: Boolean = true
    println("User: $name, Age: $age, Score: $score, Active: $isActive")
}

This code declares variables and prints them. Kotlin infers types automatically, but you can specify them explicitly (e.g., `score: Double`).

Step 4: Create Functions

Functions in Kotlin are concise and powerful. Here’s an example with parameters and return values:

fun greetUser(name: String, greeting: String = "Hello"): String {
    return "$greeting, $name!"
}

fun main() {
    val message = greetUser("Midway")
    println(message) // Output: Hello, Midway!
    println(greetUser("Android", "Welcome")) // Output: Welcome, Android!
}

Key Features:

  • Default parameters (e.g., `greeting: String = "Hello"`).
  • Return type specified after the function (e.g., `: String`).
  • Single-expression functions can omit curly braces (e.g., `fun square(num: Int) = num * num`).

Step 5: Master Null Safety

Kotlin’s null safety prevents null pointer exceptions. Variables are non-null by default; use `?` for nullable types.

fun main() {
    val name: String = "Midway" // Non-null
    // val name: String = null // Error: Null cannot be a value of a non-null type

    var nullableName: String? = null // Nullable
    println(nullableName?.length) // Safe call: Outputs null if nullableName is null
    println(nullableName ?: "Unknown") // Elvis operator: Default value if null

    nullableName = "Android"
    println(nullableName!!.length) // Non-null assertion (use cautiously)
}

Tools: `?.` (safe call), `?:` (Elvis operator), `!!` (non-null assertion). These features make Kotlin safer than Java.

Step 6: Use Control Flow

Kotlin’s control flow (if, when, loops) is expressive. Example:

fun main() {
    val score = 85
    val grade = when (score) {
        in 90..100 -> "A"
        in 80..89 -> "B"
        else -> "C"
    }
    println("Score: $score, Grade: $grade")

    for (i in 1..3) {
        println("Loop $i")
    }
}

The `when` expression is like a switch statement but more powerful. Loops support ranges (e.g., `1..3`).

Best Practices for Kotlin

  • Prefer `val` over `var`: Immutable variables are safer.
  • Use Null Safety: Avoid `!!` unless certain the value is non-null.
  • Leverage Extensions: Add functionality to classes (e.g., `fun String.addExclamation() = "$this!"`).
  • Keep Code Concise: Use Kotlin’s features like default parameters and single-expression functions.
  • Test in REPL: Use Android Studio’s Kotlin REPL for quick prototyping.

Troubleshooting Common Issues

  • Null Errors: Check for `null` using `?.` or `?:` instead of `!!`.
  • Build Fails: Ensure Kotlin plugin and Gradle are up-to-date in Android Studio.
  • Syntax Errors: Use Android Studio’s code completion and error highlighting.

Refer to Kotlin’s official docs or contact us for help.
