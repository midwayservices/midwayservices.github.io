Getting Started with HTML & CSS: Build Your First Webpage

Published: September 10, 2025 | 10 min read

HTML (HyperText Markup Language) is the backbone of every webpage, defining its structure, while CSS (Cascading Style Sheets) adds style and visual appeal. In this beginner-friendly tutorial, we’ll guide you through creating your first webpage using HTML and CSS. You’ll learn the basic structure, essential elements, styling techniques, and responsive design principles. No prior experience needed—just a text editor and enthusiasm!

Why Learn HTML & CSS?

HTML and CSS are the foundation of web development. HTML organizes content (text, images, links), while CSS controls layout, colors, and fonts. Together, they enable you to create static websites that look great on any device. This tutorial is perfect for students, hobbyists, or anyone starting their web development journey.

What You’ll Learn:

  • Creating a basic HTML structure.
  • Styling webpages with CSS.
  • Making your site responsive for mobile devices.
  • Best practices for clean, accessible code.

Step 1: Set Up Your Environment

You don’t need fancy tools to start—just a text editor and a browser. Recommended tools:

  • Text Editor: Visual Studio Code (free, with HTML/CSS extensions).
  • Browser: Google Chrome or Firefox (use Developer Tools to inspect elements).
  • Optional: Live Server extension for VS Code to preview changes instantly.

Create a folder (e.g., `my-first-webpage`) with two files: `index.html` and `style.css`. Link them in HTML with ``.

Step 2: Create the HTML Structure

HTML uses tags to define content. Below is a basic webpage structure:

<!DOCTYPE html>
<html lang="en">
<head>
  <meta charset="UTF-8">
  <meta name="viewport" content="width=device-width, initial-scale=1.0">
  <title>My First Webpage</title>
  <link rel="stylesheet" href="style.css">
</head>
<body>
  <header>
    <h1>Welcome to My Webpage</h1>
  </header>
  <main>
    <section>
      <h2>About Me</h2>
      <p>Hi, I’m learning web development with Midway Learning!</p>
    </section>
    <section>
      <h2>Contact</h2>
      <p>Reach me at <a href="mailto:example@email.com">example@email.com</a>.</p>
    </section>
  </main>
  <footer>
    <p>© 2025 My Webpage</p>
  </footer>
</body>
</html>

Save this as `index.html` and open it in a browser to see a basic webpage.

Step 3: Style Your Webpage with CSS

CSS makes your webpage visually appealing. Create `style.css` and add the following styles:

body {
  font-family: 'Roboto', Arial, sans-serif;
  margin: 0;
  line-height: 1.6;
  background: #f4f4f9;
  color: #333;
}

header {
  background: #1a1a1a;
  color: #fff;
  text-align: center;
  padding: 1rem;
}

h1 {
  font-size: 2.5rem;
  margin: 0;
}

main {
  max-width: 800px;
  margin: 2rem auto;
  padding: 1rem;
  background: #fff;
  border-radius: 8px;
}

h2 {
  color: #00aaff;
}

a {
  color: #00aaff;
  text-decoration: none;
}

a:hover {
  text-decoration: underline;
}

footer {
  text-align: center;
  padding: 1rem;
  background: #1a1a1a;
  color: #fff;
}

This CSS sets a clean layout with a dark header/footer, a centered content area, and a modern color scheme. Save and refresh your browser to see the changes.

Tip: Use Google Fonts to include `'Roboto'` by adding `` to your HTML ``.

Step 4: Make Your Webpage Responsive

Ensure your webpage looks good on all devices using CSS media queries:

@media (max-width: 768px) {
  h1 {
    font-size: 2rem;
  }

  main {
    margin: 1rem;
    padding: 1rem;
  }
}

@media (max-width: 480px) {
  h1 {
    font-size: 1.8rem;
  }
}

Add this to `style.css`. It adjusts font sizes and margins for tablets and phones. Test by resizing your browser or using Chrome’s Developer Tools (F12 > Toggle Device Toolbar).

Best Practices for HTML & CSS

  • Use Semantic HTML: Tags like `
    `, `
    `, and `
    ` improve accessibility and SEO.
  • Keep CSS DRY: Reuse styles with classes to avoid repetition.
  • Ensure Accessibility: Use alt text for images, sufficient color contrast, and ARIA attributes (e.g., `aria-label`).
  • Validate Code: Use W3C validators for HTML and CSS to catch errors.
  • Optimize Performance: Minify CSS and use compressed images.

Test your site with tools like Lighthouse (in Chrome DevTools) to check accessibility and performance.
