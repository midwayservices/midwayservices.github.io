React Basics: Building Interactive Web Interfaces

Published: August 30, 2025 | 12 min read

React is a powerful JavaScript library for building user interfaces, known for its component-based architecture and efficient virtual DOM. Developed by Facebook, it’s widely used for creating dynamic web applications. In this beginner-friendly tutorial, we’ll cover React basics—components, JSX, props, and state—with practical examples to help you build your first React app.

Why Learn React?

React simplifies web development by breaking UIs into reusable components. Its key benefits include:

  • Component-Based: Build modular, reusable UI pieces.
  • Virtual DOM: Optimizes updates for fast performance.
  • JSX: Write HTML-like code within JavaScript for intuitive syntax.
  • Ecosystem: Supports tools like React Router and Redux for advanced apps.

This tutorial assumes basic JavaScript knowledge (see our Introduction to JavaScript). Let’s get started!

Step 1: Set Up Your Environment

To start with React, you need Node.js and a code editor:

  • Node.js: Download from nodejs.org (LTS version, e.g., 20.x as of 2025).
  • Code Editor: Visual Studio Code with extensions like Prettier and ESLint.
  • Browser: Chrome or Firefox for debugging with React Developer Tools.

Create a new React project using Create React App:

npx create-react-app my-first-react-app
cd my-first-react-app
npm start

This sets up a development server at `http://localhost:3000`. The project includes `src/App.js` for your main component.

Step 2: Create Your First React Component

React apps are built with components—functions or classes that return UI elements. Here’s a modern "Hello World" example using React 18:

// src/index.js
import React from 'react';
import { createRoot } from 'react-dom/client';
import App from './App';

const root = createRoot(document.getElementById('root'));
root.render(<App />);

// src/App.js
import React from 'react';
import './App.css';

function App() {
  return (
    <div>
      <h1>Hello, React!</h1>
      <p>Welcome to Midway Learning's React tutorial!</p>
    </div>
  );
}

export default App;

Save these files, run `npm start`, and see the output in your browser. The `createRoot` API (React 18) replaces the deprecated `ReactDOM.render`. Ensure your `index.html` has a `

` for rendering.

Step 3: Master JSX

JSX is a syntax extension that lets you write HTML-like code in JavaScript. It’s compiled to JavaScript by tools like Babel. Example:

import React from 'react';

function Welcome() {
  const name = 'Midway';
  return (
    <section>
      <h2>Welcome, {name}!</h2>
      <p>Learn React with us!</p>
    </section>
  );
}

export default Welcome;

Key JSX Rules:

  • Return a single element (wrap multiple elements in a `
    ` or `<>`).
  • Use camelCase for attributes (e.g., `className` instead of `class`).
  • Embed JavaScript expressions in `{}`.

Import this component into `App.js` to render it.

Step 4: Use Props for Dynamic Data

Props (properties) pass data to components. Here’s an example:

import React from 'react';

function Greeting(props) {
  return <h2>Hello, {props.name}!</h2>;
}

function App() {
  return (
    <div>
      <Greeting name="Midway" />
      <Greeting name="React" />
    </div>
  );
}

export default App;

Props make components reusable. Pass data like `name` to customize output.

Step 5: Manage State with Hooks

State allows components to manage dynamic data. Use the `useState` hook in functional components:

import React, { useState } from 'react';

function Counter() {
  const [count, setCount] = useState(0);

  return (
    <div>
      <p>Count: {count}</p>
      <button onClick={() => setCount(count + 1)}>Increment</button>
    </div>
  );
}

export default Counter;

The `useState` hook declares `count` (state) and `setCount` (updater function). Clicking the button updates the state, triggering a re-render.

Best Practices for React

  • Keep Components Small: Break UIs into reusable components.
  • Use Functional Components: Prefer functions with hooks over class components.
  • Ensure Accessibility: Use ARIA attributes (e.g., `aria-label`) for interactive elements.
  • Optimize Performance: Avoid unnecessary re-renders with `useMemo` or `useCallback`.
  • Lint Code: Use ESLint to catch errors and enforce consistency.

Test with React Developer Tools and Lighthouse for performance and accessibility.

Troubleshooting Common Issues

  • Component Not Rendering: Check for correct imports and `id="root"` in `index.html`.
  • JSX Errors: Ensure single parent element and valid syntax.
  • State Not Updating: Use `setState` correctly; avoid mutating state directly.

Debug with React Developer Tools or contact us for help.
